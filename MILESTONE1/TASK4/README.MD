Task 4:   AI & Automation in Development: Redefining the Programmer's Craft
Software development is experiencing its most profound transformation since the transition to high-level languages. AI pair programmers and intelligent automation are no longer experimental—they are reshaping how code is written, tested, deployed, and maintained.
This session moves beyond the hype to provide a grounded, critical examination of AI-assisted development. Drawing on empirical studies and production experience, we analyze the true productivity gains of tools like GitHub Copilot, the emerging paradigm of "vibe coding," and the shift from developer-as-writer to developer-as-orchestrator.
We extend our lens beyond code generation to examine AI-driven testing, predictive DevOps, and automated documentation. Through case studies and a proposed framework for human-AI collaboration, we argue that the programmer's core competency is evolving from implementation to evaluation—from writing correct code to recognizing correct code among machine-generated alternatives.
Crucially, we confront uncomfortable questions: Are we augmenting ability or atrophying skill? What are the security and legal implications of AI-generated code? Who benefits from this transition, and whose labor is displaced?
This presentation offers no simplistic predictions about AI replacing developers. Instead, it provides a nuanced roadmap for navigating the transition from coder to curator, equipping attendees with practical heuristics, risk awareness, and a coherent mental model for the AI-augmented engineering organization.
Key takeaways: Empirical assessment of current tools 
• Framework for human-AI collaboration 
• Critical risks and mitigation strategies
 • Future trajectories for 2028 
• Actionable adaptations for teams and curricula.
